## 2026-07-24 — Git-backed Zoro presence implemented

- **Components:** Ideas Hub coordination, Zoro, Architect, runtime generation, operational logging
- **Authority:** Kofi explicitly requested implementation directly on `kofiarhin/ideahub` `main`.
- **Change:** Added `coordination/presence/README.md` and `coordination/presence/zoro.json`; defined a 60-minute advisory lease, current-blob concurrency, stale-state interpretation, conflict handling, release behavior, and a narrow standing direct-`main` exception only for Zoro presence transitions.
- **Zoro behavior:** Zoro instruction version and runtime are `1.3.0`; Zoro reads presence before meaningful implementation, establishes or reconciles a session, renews near expiry or on activity changes, reports waiting/blockers, and releases the session when work ends.

Presence remains advisory and Architect is read-only without separate authority.

- **Logging:** Routine presence start, renewal, waiting, blocked, release, and stale observations are excluded from repository-activity logs; protocol or permission changes remain system changes.
- **Runtime state:** `runtime/manifest.json` records Zoro `1.3.0`, Architect `1.2.0`, and refreshed canonical source fingerprints. Runtime template and output writes returned identical content blob SHAs for each agent.
